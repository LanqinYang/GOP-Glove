# Rationale, Aim and Objectives (RAO)

## Rationale（为什么要做）
- 自制柔性/应变传感器在真实佩戴中存在显著不稳定：材料滞回与蠕变、温漂、绑带与装配差异、ADC/供电噪声、跨用户/跨会话分布偏移。这使得常规方法在端侧设备上难以稳定复用。
- TinyML 部署受算力与存储强约束，高精度深度模型常难以在微控制器上落地，而可部署的传统模型又往往难以跨主体泛化。
- 需要一条可落地的端到端路径，使“不稳定的 DIY 传感器”也能“用起来”，并在嵌入式设备上稳定、低延迟地工作。本文以混合范式 ADANN_LightGBM（域鲁棒特征 + 轻量树分类）作为核心方法。

## Aim（最终要达成什么）
构建并验证一套面向不稳定 DIY 传感器的端到端 BSL 手势识别系统，兼顾跨用户鲁棒泛化与 Arduino Nano 33 BLE Sense Rev2 上的稳定低延迟实时推理，核心方法为 ADANN_LightGBM。

## Objectives（为达成 Aim 的步骤）
1) 传感器表征与问题刻画：量化不稳定性（SNR、漂移率、滞回、跨会话/跨用户一致性），形成工程约束与设计依据。
2) 数据采集与规范：完成五通道 50 Hz 采集与命名规范，保证 LOSO 可用主体信息。
3) 预处理与规范化：统一重采样到 100×5；轻量数据增强（加噪/时间扭曲/幅度缩放）；训练集拟合 StandardScaler 并应用到各切分。
4) 基线与统一训练：在统一管线上训练六模型（1D_CNN / Transformer_Encoder / XGBoost / LightGBM / ADANN / ADANN_LightGBM），使用 Optuna（TPE + MedianPruner）进行超参搜索，并以 LOSO 评估跨用户泛化。说明：HPO 具随机性，固定随机种子可减小方差，但结果存在合理波动，应报告方差/置信区间而非承诺“一键复跑完全一致”。
5) 核心方法验证（创新点）：重点比较 ADANN_LightGBM 与最佳基线（如 1D_CNN 或 ADANN），在 Arduino-LOSO 场景展示准确率优势（当前典型结果：其他≈72%，CNN/ADANN≈78%，ADANN_LightGBM≈82%），并辅以统计检验与消融（增强/试验数/epochs、Kalman 去除）。
6) TinyML 部署：在 Arduino 模式下应用幅度剪枝（30→60% 多项式）与 INT8 量化；自动导出头文件并记录模型/头文件大小与资源约束满足情况。
7) 实时演示（应用证据）：实现在线滑窗分类（窗=100，步=10，2 分钟 × 3 轮），对比 ADANN_LightGBM 与最佳基线的在线准确率、平均延迟与方差、抖动与误报率、姿态保持稳定性。
8) 工程与可追溯性：提供运行脚本与固定随机种子，记录关键版本与配置；承认 HPO 的随机性，采用多次运行与统计区间报告结果稳健性。
